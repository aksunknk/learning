🧵 通しプロジェクト

同一アプリ TaskBoard を、HTML → JS → API/DB → React → Docker → CI/CD → セキュリティ点検まで一本で繋ぐ縦糸演習です。他チャプターは辞書、この章は組み立て図として使います。

🟢 初級編 — 1 つの画面を、手元だけで動かす

成果物 TaskBoard を定義し、HTML/CSS → JavaScript → Git まで。サーバー無しで「使える UI」を先に作ります。

01

縦糸プロジェクトとは — TaskBoard を一本の糸にする

▼

💡 章は「部品」、この章は「組み立て図」

他チャプターで学んだ技術を、同一アプリ TaskBoard（タスク管理）に順に接続します。 ゴールはフレームワーク全部を同時に覚えることではなく、「今どの層を触っているか」を常に説明できることです。キャップストーンの直前に、結合の地図を頭に入れるための章です。

成果物の定義（スコープを狭く固定する）

機能を増やしすぎると縦糸が切れます。v1 は次の 4 つだけに限定します。

03

Step B — JavaScript で操作し localStorage に残す

▼

💡 サーバー前に「状態」の形を確定する

タスク配列を唯一の状態源にし、描画関数で DOM を更新します。 永続化はとりあえず localStorage。このデータ形（id / title / done）を後で API・DB・React まで引きずります。形を途中で大きく変えないことが、縦糸を切らないコツです。

🔗 関連チャプター

このステップの詳細学習: javascript（JavaScript）

状態と描画の分離

イベントハンドラは状態を変え、最後に render() を呼ぶ——React に進んでも同じ発想です。

JavaScript 
const KEY = "taskboard.v1";
/** @type {{id:string,title:string,done:boolean}[]} */
let tasks = JSON.parse(localStorage.getItem(KEY) || "[]");

const form = document.querySelector("#task-form");
const input = document.querySelector("#task-title");
const list = document.querySelector("#task-list");
const empty = document.querySelector("#empty");

function save() {
  localStorage.setItem(KEY, JSON.stringify(tasks));
}

function render() {
  list.innerHTML = "";
  empty.hidden = tasks.length > 0;
  for (const t of tasks) {
    const li = document.createElement("li");
    if (t.done) li.classList.add("done");
    li.innerHTML = ""; // XSS 回避のため textContent で組み立て
    const check = document.createElement("input");
    check.type = "checkbox";
    check.checked = t.done;
    check.addEventListener("change", () => {
      t.done = check.checked;
      save();
      render();
    });
    const span = document.createElement("span");
    span.textContent = t.title;
    const del = document.createElement("button");
    del.type = "button";
    del.textContent = "削除";
    del.addEventListener("click", () => {
      tasks = tasks.filter((x) => x.id !== t.id);
      save();
      render();
    });
    li.append(check, span, del);
    list.append(li);
  }
}

form.addEventListener("submit", (e) => {
  e.preventDefault();
  const title = input.value.trim();
  if (!title) return;
  tasks.push({ id: crypto.randomUUID(), title, done: false });
  input.value = "";
  save();
  render();
});

render();

Step C-2 — FastAPI + DB で永続化する

▼

💡 ブラウザの外に「真実」を移す

localStorage をやめるのではなく、サーバーを真実源にします。 テーブル tasks を作り、契約どおりの CRUD を実装します。最初は SQLite で十分です。マイグレーションの習慣だけ先に付けます。

🔗 関連チャプター

このステップの詳細学習: python（Python） / database（データベース） / webapi（Web/API）

Python 
# 概念実装（詳細は python / database / webapi 章へ）
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field

app = FastAPI()

class TaskIn(BaseModel):
    title: str = Field(min_length=1, max_length=100)

class TaskPatch(BaseModel):
    title: str | None = Field(default=None, min_length=1, max_length=100)
    done: bool | None = None

@app.get("/api/tasks")
def list_tasks():
    return {"items": db_list()}

@app.post("/api/tasks", status_code=201)
def create_task(body: TaskIn):
    return db_create(body.title)

@app.patch("/api/tasks/{task_id}")
def patch_task(task_id: str, body: TaskPatch):
    task = db_update(task_id, body)
    if not task:
        raise HTTPException(404, detail={"error": {"code": "NOT_FOUND", "message": "task not found"}})
    return task

@app.delete("/api/tasks/{task_id}", status_code=204)
def delete_task(task_id: str):
    if not db_delete(task_id):
        raise HTTPException(404)
    return None
