言語仕様①：内包表記とジェネレータ（第3章・出題比率17.5%）

✓

🎯 なぜ内包表記とジェネレータが最頻出なのか？

第3章「Pythonの言語仕様」は全40問中7問（17.5%）を占める最重要セクションです。中でも内包表記とジェネレータは「Pythonic（Python らしい）」コードの代名詞であり、実務コードのほぼ全てに登場します。forループで5行書くコードを1行に凝縮する内包表記は、可読性とパフォーマンスを同時に改善します。一方、ジェネレータは100万件のデータをメモリに乗せずに逐次処理する「遅延評価」の仕組みです。

内包表記の基本と応用（リスト・辞書・集合）
# ============================================================
# 内包表記 ── forループの「Pythonicな書き換え」
# ============================================================

# --- 基本のリスト内包表記 ---
# 従来のforループ（初心者が書きがちなパターン）
squares_old = []
for i in range(10):
    squares_old.append(i ** 2)

# 内包表記に変換（1行で完結）
squares = [i ** 2 for i in range(10)]
# => [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

# --- 条件付き内包表記（if でフィルタリング） ---
# 偶数だけの二乗を取得する
even_squares = [i ** 2 for i in range(10) if i % 2 == 0]
# => [0, 4, 16, 36, 64]

# ⚠️ 試験頻出ポイント: if-else は「前」に置く
# 「偶数ならそのまま、奇数なら-1を掛ける」
result = [x if x % 2 == 0 else -x for x in range(5)]
# => [0, -1, 2, -3, 4]
# ❌ よくある間違い: [x for x in range(5) if x % 2 == 0 else -x]
# SyntaxError になる。if-else（三項演算子）は for の前に書く。

# --- 辞書内包表記 ---
# キーと値のペアから辞書を一発で作る
word = "hello"
char_count = {c: word.count(c) for c in set(word)}
# => {'h': 1, 'e': 1, 'l': 2, 'o': 1}

# --- 集合内包表記 ---
# 重複を自動的に排除してくれる
lengths = {len(w) for w in ["apple", "banana", "kiwi", "fig"]}
# => {3, 4, 5, 6}

# --- ネストした内包表記 ---
# 行列（2次元リスト）を1次元にフラット化する
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flat = [x for row in matrix for x in row]
# => [1, 2, 3, 4, 5, 6, 7, 8, 9]
# ⚠️ 読む順番: 外側の for が先、内側の for が後ろ
ジェネレータ ── メモリを食わない遅延評価
# ============================================================
# ジェネレータ ── yield で「一つずつ返す」関数
# ============================================================

def fibonacci(n):
    """フィボナッチ数列をn個生成するジェネレータ。
    
    ⚠️ 試験のポイント:
    - return で値を返すのではなく yield で「一時停止しながら」返す
    - 呼び出すたびに前回の yield の次から実行が再開される
    - 関数を呼んだ時点ではコードは実行されない（遅延評価）
    """
    a, b = 0, 1
    for _ in range(n):
        yield a          # ← ここで一時停止して値を返す
        a, b = b, a + b  # ← next() が呼ばれたらここから再開

# ジェネレータを使う
gen = fibonacci(10)
print(type(gen))  # => 

# next() で1つずつ取り出せる
print(next(gen))  # => 0
print(next(gen))  # => 1
print(next(gen))  # => 1

# for ループで残りを全て取り出す
for num in gen:
    print(num, end=" ")  # => 2 3 5 8 13 21 34

# --- ジェネレータ式（内包表記のジェネレータ版） ---
# リスト内包表記: [...] → メモリに全て格納
# ジェネレータ式: (...) → 1つずつ生成（メモリ節約）
import sys

list_comp = [i ** 2 for i in range(1000000)]
gen_expr  = (i ** 2 for i in range(1000000))

print(sys.getsizeof(list_comp))  # => 約 8.4MB
print(sys.getsizeof(gen_expr))   # => 約 200B（！）
# ⚠️ 巨大データを処理するときはジェネレータ式を使う

💼 Tips：実務で遭遇する現場の勘所

実務では「内包表記は3行以内に収まるなら使う、それ以上複雑になるならforループに戻す」が暗黙のルールです。ネストが深い内包表記はコードレビューで確実にリジェクトされます。また、ジェネレータはログファイルの処理やAPIのページネーション（次のページを取得しながら処理する）で大活躍します。yield from を使ったサブジェネレータの委譲も試験で問われることがあるので、書籍の該当箇所を確認しておきましょう。

言語仕様②：デコレータとコンテキストマネージャ（第3章）

✓

🎯 なぜデコレータとコンテキストマネージャが重要なのか？

デコレータは「関数を変更せずに機能を追加する」仕組みです。Webフレームワーク（Flask, FastAPI）の @app.route() や、Django の @login_required など、実務のPythonコードは至る所にデコレータが使われています。コンテキストマネージャは with 文と組み合わせて「リソースの確保と解放を自動化」する仕組みで、ファイル操作、DB接続、ロック管理に不可欠です。

デコレータ ── 関数を「ラップ」する高階関数
# ============================================================
# デコレータ ── @構文の裏側で何が起きているか
# ============================================================
import functools
import time

def timer(func):
    """関数の実行時間を計測するデコレータ。
    
    ⚠️ 試験のポイント:
    - @functools.wraps(func) を忘れると、元の関数名やdocstringが
      wrapper のものに上書きされてしまう（デバッグ時に困る）
    - デコレータは「関数を受け取り、関数を返す関数」
    """
    @functools.wraps(func)  # ← 元関数のメタ情報を保持
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)  # 元の関数を実行
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timer  # ← process_data = timer(process_data) と同義
def process_data(n):
    """大量データを処理する関数"""
    return sum(i ** 2 for i in range(n))

result = process_data(1_000_000)
# => process_data took 0.1234s

# デコレータなしで書くと…
# process_data = timer(process_data)  # これと @timer は完全に同じ

# --- 引数付きデコレータ ---
def repeat(n):
    """関数をn回繰り返すデコレータ（デコレータファクトリ）"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)  # ← repeat(3) がデコレータを返す → それが greet に適用される
def greet(name):
    print(f"Hello, {name}!")

greet("Python")
# => Hello, Python! (3回出力)
コンテキストマネージャ ── with 文の裏側
# ============================================================
# コンテキストマネージャ ── リソース管理の自動化
# ============================================================
from contextlib import contextmanager

# --- クラスベースのコンテキストマネージャ ---
class DatabaseConnection:
    """DB接続を安全に管理するコンテキストマネージャ。
    
    ⚠️ 試験のポイント:
    - __enter__ の戻り値が as の変数に入る
    - __exit__ は例外の有無に関わらず「必ず」呼ばれる
    - __exit__ で True を返すと例外を握りつぶす（通常は非推奨）
    """
    def __init__(self, db_name):
        self.db_name = db_name
    
    def __enter__(self):
        print(f"DB '{self.db_name}' に接続")
        self.conn = {"connected": True}  # 実際はDB接続オブジェクト
        return self.conn  # ← as conn の conn に入る
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        print(f"DB '{self.db_name}' を切断")
        self.conn["connected"] = False
        return False  # ← 例外があれば再送出する

with DatabaseConnection("production") as conn:
    print(f"接続状態: {conn['connected']}")
# => DB 'production' に接続
# => 接続状態: True
# => DB 'production' を切断  ← 例外が起きても必ず実行される

# --- @contextmanager を使った簡潔な書き方 ---
@contextmanager
def timer_context(label):
    """処理時間を計測するコンテキストマネージャ（ジェネレータ版）。
    
    ⚠️ yield の前が __enter__、yield の後が __exit__ に相当する
    """
    start = time.perf_counter()
    try:
        yield  # ← with ブロックの中身がここで実行される
    finally:
        elapsed = time.perf_counter() - start
        print(f"[{label}] {elapsed:.4f}s")

with timer_context("データ処理"):
    total = sum(range(10_000_000))
# => [データ処理] 0.3456s

💼 Tips：実務で遭遇する現場の勘所

実務では @contextmanager を使ったジェネレータ版の方が圧倒的に多用されます（クラスベースは冗長なため）。デコレータは複数を重ねる（スタックする）ことができますが、適用順序は下から上であることに注意。@decorator_a、@decorator_b の順に書くと、先に decorator_b が適用され、その結果に decorator_a が適用されます。これは試験でもよく問われるポイントです。

クラスと特殊メソッド（第4章・出題比率7.5%）

✓

🎯 なぜ特殊メソッド（ダンダーメソッド）を理解する必要があるのか？

Pythonのクラスは特殊メソッド（__init__, __repr__, __eq__ 等）を定義することで、組み込み型と同じように振る舞うオブジェクトを作れます。len(obj) が呼べるのは __len__ が定義されているから、print(obj) で意味のある出力が出るのは __repr__ があるからです。試験ではこれらの特殊メソッドの「どれがどの操作に対応するか」が問われます。

特殊メソッドで「自分だけのデータ型」を作る
# ============================================================
# 実践的なクラス設計 ── 特殊メソッドのフル活用
# ============================================================
from functools import total_ordering

@total_ordering  # __eq__ と __lt__ だけで全ての比較演算子を自動生成
class Money:
    """通貨を扱うクラス。
    
    ⚠️ 試験のポイント:
    - __repr__: 開発者向けの文字列表現（eval()で再現可能が理想）
    - __str__: ユーザー向けの文字列表現（print()で使われる）
    - __add__: + 演算子のオーバーロード
    - __eq__: == 演算子のオーバーロード
    - __lt__: < 演算子のオーバーロード
    - __hash__: 辞書のキーやset の要素にするために必要
    """
    def __init__(self, amount, currency="JPY"):
        self.amount = amount
        self.currency = currency
    
    def __repr__(self):
        return f"Money({self.amount}, '{self.currency}')"
    
    def __str__(self):
        if self.currency == "JPY":
            return f"¥{self.amount:,}"
        return f"${self.amount:,.2f}"
    
    def __add__(self, other):
        if self.currency != other.currency:
            raise ValueError(f"通貨が異なります: {self.currency} vs {other.currency}")
        return Money(self.amount + other.amount, self.currency)
    
    def __eq__(self, other):
        return self.amount == other.amount and self.currency == other.currency
    
    def __lt__(self, other):
        if self.currency != other.currency:
            raise ValueError("異なる通貨は比較できません")
        return self.amount < other.amount
    
    def __hash__(self):
        return hash((self.amount, self.currency))

# 使用例
price = Money(1500)
tax = Money(150)
total = price + tax
print(total)       # => ¥1,650
print(repr(total))  # => Money(1650, 'JPY')
print(price < total)  # => True

# __hash__ があるので set や dict のキーに使える
prices = {Money(100), Money(200), Money(100)}
print(len(prices))  # => 2 (重複が除去される)

# --- プロパティ（@property） ---
class Circle:
    """半径から面積を計算するクラス。
    
    ⚠️ @property はゲッターとして機能し、
    属性アクセスの形式（.area）でメソッドを呼べる
    """
    import math
    
    def __init__(self, radius):
        self._radius = radius
    
    @property
    def radius(self):
        return self._radius
    
    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("半径は0以上でなければなりません")
        self._radius = value
    
    @property
    def area(self):
        import math
        return math.pi * self._radius ** 2

c = Circle(5)
print(f"面積: {c.area:.2f}")  # => 面積: 78.54
c.radius = 10                  # setter が呼ばれる
# c.radius = -1               # => ValueError

💼 Tips：実務で遭遇する現場の勘所

実務では dataclasses モジュールの @dataclass デコレータを使えば、__init__, __repr__, __eq__ を自動生成できるため、ボイラープレートを大幅に削減できます。ただし dataclasses は試験範囲外なので、試験対策としては手動で特殊メソッドを書く方法を必ず理解しておきましょう。@total_ordering は functools モジュールの便利機能で、__eq__ と1つの比較メソッドだけで全比較演算子を生成してくれます。

タイプヒントとコーディング規約 PEP 8（第2・5章）

✓

🎯 なぜ型を書くのか？ Pythonは動的型付け言語なのに

Pythonは実行時に型チェックをしないため、型ヒントは「ドキュメントとしての役割」と「静的解析ツール（mypy等）での事前チェック」が主な目的です。試験では第5章から2問出題されますが、5.2 mypyは除外されているため、型ヒントの「書き方」に集中すればOKです。PEP 8（第2章・2問）は「Pythonコードのスタイルガイド」であり、インデント幅や命名規則などの「お作法」を問います。

タイプヒント ── 型を「注釈」として書く
# ============================================================
# タイプヒント ── Python 3.9+ の現代的な書き方
# ============================================================
from typing import Optional, Union, Callable

# --- 基本的な型ヒント ---
name: str = "Python"
age: int = 35
price: float = 19.99
is_active: bool = True

# --- 関数の型ヒント ---
def greet(name: str, times: int = 1) -> str:
    """引数と戻り値に型を付ける。
    
    ⚠️ 試験のポイント:
    - -> で戻り値の型を指定する
    - 型ヒントは実行時には無視される（強制力はない）
    - あくまで「ドキュメント」と「ツール向けの情報」
    """
    return (f"Hello, {name}! " * times).strip()

# --- コレクション型 ---
# Python 3.9+ では組み込み型をそのまま使える
scores: list[int] = [90, 85, 78]
config: dict[str, str] = {"host": "localhost", "port": "5432"}
unique_ids: set[int] = {1, 2, 3}
coordinates: tuple[float, float] = (35.6762, 139.6503)

# --- Optional と Union ---
# Optional[X] は Union[X, None] の短縮形
def find_user(user_id: int) -> Optional[dict]:
    """ユーザーが見つからない場合は None を返す"""
    if user_id == 1:
        return {"name": "Alice"}
    return None

# Union: 複数の型を許容する
def process(data: Union[str, bytes]) -> str:
    if isinstance(data, bytes):
        return data.decode("utf-8")
    return data

# --- Callable（関数の型） ---
def apply_twice(func: Callable[[int], int], value: int) -> int:
    """関数を2回適用する高階関数"""
    return func(func(value))

result = apply_twice(lambda x: x * 2, 3)  # => 12
PEP 8 ── 頻出の命名規則とスタイル
# ============================================================
# PEP 8 ── 試験で問われるコーディング規約
# ============================================================

# ✅ 変数名・関数名: スネークケース（snake_case）
user_name = "Alice"
def calculate_total_price(): pass

# ✅ クラス名: キャメルケース（CamelCase / PascalCase）
class ShoppingCart: pass
class HTTPSConnection: pass  # 頭字語は全て大文字のまま

# ✅ 定数: 全て大文字のスネークケース（UPPER_SNAKE_CASE）
MAX_RETRY_COUNT = 3
DATABASE_URL = "postgresql://localhost/mydb"

# ✅ プライベート属性: アンダースコア1つで始める
class User:
    def __init__(self):
        self._internal_state = {}   # 慣習的プライベート

# ✅ インデントは半角スペース4つ（タブは使わない）

正規表現（re モジュール）── パターンマッチングの決定版
# ============================================================
# re モジュール ── 各関数の違いを正確に理解する
# ============================================================
import re

text = "電話番号は 03-1234-5678 または 090-9876-5432 です"

# --- re.search(): 文字列全体から最初のマッチを探す ---
match = re.search(r'\d{2,3}-\d{4}-\d{4}', text)
if match:
    print(match.group())   # => 03-1234-5678
    print(match.start())   # => 5 (マッチ開始位置)

# --- re.match(): 文字列の「先頭」からのみマッチ ---
# ⚠️ 試験最頻出: search と match の違い
result = re.match(r'\d+', text)
print(result)  # => None (先頭は「電」なのでマッチしない)

result = re.match(r'\d+', "123abc")
print(result.group())  # => 123 (先頭が数字なのでマッチ)

# --- re.findall(): 全てのマッチをリストで返す ---
phones = re.findall(r'\d{2,3}-\d{4}-\d{4}', text)
print(phones)  # => ['03-1234-5678', '090-9876-5432']

# --- re.sub(): マッチした部分を置換する ---
masked = re.sub(r'\d{4}-\d{4}', 'XXXX-XXXX', text)
print(masked)  # => 電話番号は 03-XXXX-XXXX または 090-XXXX-XXXX です

# --- グループ化とキャプチャ ---
pattern = r'(\d{2,3})-(\d{4})-(\d{4})'
m = re.search(pattern, text)
if m:
    print(m.group(0))  # => 03-1234-5678 (全体)
    print(m.group(1))  # => 03 (市外局番)
    print(m.group(2))  # => 1234
    print(m.groups())  # => ('03', '1234', '5678')

# --- コンパイル済みパターン（繰り返し使う場合に高速） ---
phone_pattern = re.compile(r'\d{2,3}-\d{4}-\d{4}')
result = phone_pattern.findall(text)

💼 Tips：実務で遭遇する現場の勘所

re.match() と re.search() の違いは試験の定番問題です。match は「先頭からのマッチ」、search は「文字列全体を走査」と覚えましょう。実務では re.compile() でパターンを事前コンパイルしておくと、ループ内で何度もマッチングする際にパフォーマンスが向上します。また、正規表現を書く際は必ず r''（raw文字列）を使い、バックスラッシュのエスケープ問題を避けてください。

データ型と collections モジュール（第9章・出題比率12.5%）

✓

🎯 なぜ collections が5問も出題されるのか？

標準の dict や list だけでは不便な場面 ── 例えば「存在しないキーにアクセスしたときにデフォルト値を返してほしい」「要素の出現回数を数えたい」「先頭への追加をO(1)でやりたい」── を解決するのがcollections モジュールです。第9章は5問（12.5%）出題され、第3章に次ぐ重要分野です。試験では各クラスの「使い分け」と「特有のメソッド」が問われます。

collections モジュール完全ガイド
# ============================================================
# collections ── 標準dict/listの「不満」を解消する特殊コンテナ
# ============================================================
from collections import defaultdict, Counter, namedtuple, deque, OrderedDict

# --- defaultdict: キーが無いときのデフォルト値を自動生成 ---
# ⚠️ 通常の dict だと KeyError になる場面で使う
word_groups = defaultdict(list)  # デフォルトは空リスト
words = ["apple", "banana", "avocado", "blueberry", "cherry"]
for w in words:
    word_groups[w[0]].append(w)  # 最初のアクセスでも KeyError にならない

print(dict(word_groups))
# => {'a': ['apple', 'avocado'], 'b': ['banana', 'blueberry'], 'c': ['cherry']}

# defaultdict(int) ならデフォルト値は 0 → カウンターとして使える
char_count = defaultdict(int)
for c in "hello world":
    char_count[c] += 1

# --- Counter: 出現頻度を数える専用クラス ---
from collections import Counter
text = "abracadabra"
freq = Counter(text)
print(freq)               # => Counter({'a': 5, 'b': 2, 'r': 2, 'c': 1, 'd': 1})
print(freq.most_common(2))  # => [('a', 5), ('b', 2)]

# ⚠️ 試験ポイント: Counter 同士の演算ができる
c1 = Counter(a=3, b=1)
c2 = Counter(a=1, b=2)
print(c1 + c2)  # => Counter({'a': 4, 'b': 3})
print(c1 - c2)  # => Counter({'a': 2})  ← 負の値は自動的に除外される

# --- namedtuple: 名前付きフィールドを持つ不変タプル ---
Point = namedtuple('Point', ['x', 'y'])
p = Point(3, 4)
print(p.x, p.y)      # => 3 4 (名前でアクセスできる)
print(p[0], p[1])     # => 3 4 (インデックスでもアクセスできる)
# p.x = 5            # => AttributeError (不変なので変更不可)

# --- deque: 両端キュー（先頭操作がO(1)） ---
# ⚠️ list の insert(0, x) は O(n) だが、deque の appendleft は O(1)
dq = deque([1, 2, 3])
dq.appendleft(0)   # 先頭に追加: deque([0, 1, 2, 3])
dq.append(4)        # 末尾に追加: deque([0, 1, 2, 3, 4])
dq.popleft()         # 先頭を削除: deque([1, 2, 3, 4])
dq.rotate(1)         # 右に1つ回転: deque([4, 1, 2, 3])

# maxlen を指定すると固定長のリングバッファになる
recent_logs = deque(maxlen=3)
recent_logs.append("log1")
recent_logs.append("log2")
recent_logs.append("log3")
recent_logs.append("log4")  # 古い log1 が自動的に消える
print(list(recent_logs))     # => ['log2', 'log3', 'log4']
enum ── 定数をクラスで安全に管理する
# ============================================================
# enum.Enum ── マジックナンバーを排除する
# ============================================================
from enum import Enum, auto

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

class Status(Enum):
    PENDING = auto()    # 自動で 1 が割り当てられる
    APPROVED = auto()   # 自動で 2
    REJECTED = auto()   # 自動で 3

# 使い方
print(Color.RED)        # => Color.RED
print(Color.RED.name)   # => 'RED'
print(Color.RED.value)  # => 1
print(Color(2))          # => Color.GREEN (値からEnum を逆引き)
print(Color['BLUE'])     # => Color.BLUE (名前からEnum を逆引き)

# ⚠️ Enum のメンバーは比較できるが、整数とは直接比較できない
print(Color.RED == Color.RED)   # => True
print(Color.RED == 1)           # => False (!) intとは等しくない

# イテレーションも可能
for c in Color:
    print(f"{c.name}: {c.value}")

💼 Tips：実務で遭遇する現場の勘所

defaultdict(list) はグルーピング処理の定石です。例えば「ユーザーIDごとの注文一覧を作る」場合、通常の dict なら if key not in d: d[key] = [] の分岐が毎回必要ですが、defaultdict なら不要になります。Counter はログ解析やテキストマイニングで頻出。deque は「直近N件のデータだけ保持する」リングバッファとして maxlen と組み合わせて使うのが実務の定番パターンです。

    """階乗を計算する。

    1
    >>> factorial(5)
    120

    Traceback (most recent call last):

# ============================================================
# urllib ── URL の解析と HTTP アクセス
# ============================================================
from urllib.parse import urlparse, urlencode, quote
from urllib.request import urlopen

# --- URL の構造分解 ---
url = "https://example.com:8080/api/users?page=2&limit=10#section1"
parsed = urlparse(url)
print(parsed.scheme)    # => https
print(parsed.hostname)  # => example.com
print(parsed.port)      # => 8080
print(parsed.path)      # => /api/users
print(parsed.query)     # => page=2&limit=10
print(parsed.fragment)  # => section1

# --- クエリ文字列の組み立て ---
params = urlencode({"q": "Python 入門", "page": 1})
print(params)  # => q=Python+%E5%85%A5%E9%96%80&page=1

# --- URL エンコーディング ---
print(quote("日本語のパス"))  # => %E6%97%A5%E6%9C%AC%E8%AA%9E...

# ============================================================
# hashlib ── ハッシュ関数（暗号の基礎）
# ============================================================
import hashlib

# --- SHA-256 ハッシュの計算 ---
message = "パスワードを直接保存してはいけません"
hash_obj = hashlib.sha256(message.encode("utf-8"))
print(hash_obj.hexdigest())
# => 固定長の16進数文字列（毎回同じ入力なら同じ出力）

# ⚠️ 試験のポイント:
# - ハッシュは「一方向関数」→ 元のデータに戻せない
# - 同じ入力からは常に同じハッシュ値が得られる
# - MD5 は脆弱なので SHA-256 以上を使う
# - パスワード保存には hashlib ではなく bcrypt 等を使うべき
#   （ただし bcrypt は試験範囲外）

# --- ファイルのハッシュ値を計算する（改ざん検知に使う） ---
def file_hash(filepath):
    h = hashlib.sha256()
    with open(filepath, "rb") as f:
        while chunk := f.read(8192):  # ← ウォルラス演算子
            h.update(chunk)
    return h.hexdigest()

💼 Tips：実務で遭遇する現場の勘所

json.dumps() と json.dump() の違い（末尾のsがstring操作かfile操作かを示す）は試験の定番問題です。CSV書き込みで newline="" を忘れるのは実務でもよくあるバグなので注意。hashlib はファイルの整合性チェックやAPIキーの比較に使いますが、パスワードの保存には使わないでください（レインボーテーブル攻撃に弱いため）。urllib は試験対策として覚えますが、実務では requests ライブラリの方がはるかに使いやすいです（ただし requests は試験範囲外）。
